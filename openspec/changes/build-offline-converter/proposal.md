# Автономный многоверсионный конвертер XML/CF

**GitHub tracking:** [epic #178](https://github.com/Untru/ibcmd-rs/issues/178), дочерние issues #179–#234.

## Зачем

Текущий `ibcmd-rs` умеет исследовать внутренние данные 1С, напрямую экспортировать MSSQL в XML и частично собирать `ConfigSave` поверх существующей совместимой базы. Этот контур накопил ценные бинарные кодеки и проверки, но не является автономным компилятором конфигурации: часть packer-ов требует исходный blob, версии XML и платформы смешаны, а CF-контейнер реализован только частично.

Новая цель — запускать чтение, запись и межверсионное преобразование XML/CF на машине, где не установлены платформа 1С, EDT, Java-sidecar или их исполняемые компоненты.

## Что меняем

- Вводим отдельное platform-independent ядро с канонической моделью, lossless native snapshot, профилями версий, диагностикой потерь и графом миграций.
- Разделяем версию платформы, XML-диалект, режим совместимости, storage layout и ревизию CF-контейнера.
- Выделяем XML/XCF, CF и существующий MSSQL-контур в адаптеры.
- Реализуем чистый Rust reader/writer для CF Format15 и Format16, включая chained blocks, nested containers, raw DEFLATE, streaming I/O и защитные лимиты.
- Сохраняем существующие reverse-engineered декодеры как legacy codecs и постепенно переводим семейства метаданных на base-free bootstrap encoding.
- Добавляем явные режимы `inspect`, `repack`, `overlay`, `bootstrap` и `convert`; capability и отчёт о потерях определяют, какой режим допустим.
- Совместимость объявляется только на основании offline fixtures и воспроизводимой test matrix.

## Жёсткие ограничения

- Release binary и стандартный CI не ищут и не запускают `ibcmd`, Designer, `1cv8`, EDT, OSGi/JVM или иные компоненты платформы.
- Кодеки должны работать только на коде и данных проекта.
- Неизвестные данные сохраняются byte-identical только в same-profile passthrough; межверсионный перенос opaque-фрагментов без правила запрещён.
- Downgrade по умолчанию fail-closed. Потеря допустима только при явной политике и всегда попадает в machine-readable report.
- Полная сборка XML → CF без base разрешается только при нулевом числе `NeedsBase`/`Unsupported` blockers.

## Не входит в первый релиз

- Гарантия поддержки всех исторических версий платформы.
- CFE/EPF/ERF как verified artifact kinds; модель должна позволять добавить их позднее.
- Байтовое совпадение с конкретным vendor compressor. Требуется semantic roundtrip и детерминированный собственный output.
- Автоматическое угадывание неизвестного профиля с разрешением записи.

## Результат

Пользователь сможет без установленной платформы:

1. исследовать и проверять CF;
2. разбирать CF в XML;
3. детерминированно перепаковывать CF без потери неизвестных записей;
4. накладывать поддержанные XML-изменения на base CF;
5. после закрытия bootstrap-capabilities собирать новый CF из XML;
6. преобразовывать поддержанные артефакты между профилями с явным отчётом о потерях.

## Влияние

- Новые crates: `ibcmd-core`, `ibcmd-xml`, `ibcmd-v8`, `ibcmd-cf`.
- Новые каталоги: `profiles/`, `compatibility/`, versioned fixtures.
- Текущий root crate остаётся CLI и legacy integration layer.
- Публичные CLI-алиасы сохраняются на переходный период, но внутри переводятся в независимые version axes.
